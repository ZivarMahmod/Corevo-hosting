# De-risk + härdning — goal-36 (100 mallar utan att krascha)

Datum: 2026-06-26 · Källa: research-runda (intern render-bron-kartläggning + extern: Shopify/WP-FSE/Webflow/Builder/Plasmic + DOMPurify/parse5 + Playwright/Chromatic + Cloudflare-Workers-docs) vägd mot koden.

> Syfte: göra goal-36 (autonom loop, ~100 vendor-HTML-mallar → render-bron, 0 FAIL) robust nog att INTE krascha eller bli falsk-grön. Hårdning att foga in INNAN loopen körs.

## ⭐ KÄRNINSIKTEN (båda spåren landar här)

**Ingen best-in-class (Shopify / WP-FSE / Webflow / Builder / Plasmic) renderar rå vendor-HTML som live-yta. De KOMPILERAR designen till ett typat nod-träd vid IMPORT — renderar trädet sen.** Vendor-HTML = fryst ställning; bara deklarerade, *typade* "hål" är redigerbara/injicerbara. Det är arkitekturen som gör "100 mallar, 0 krascher" möjlig.

→ Corevos render-bron gör redan HALVA detta (`html-react-parser` → React-träd, edge-säkert, typade regioner via manifest). Det som FATTAS för skala = (a) flytta parse/validering till IMPORT (inte per-render), (b) lasta HTML utanför JS-bundlen, (c) mekanisk verify per mall. **Vi skriver INTE om render-motorn — vi adopterar principerna i den befintliga.**

## ✅ Det vi redan gör bra (behåll)

- render-bron (`lib/sajtbyggare/render-bridge.tsx`, ~12 rader) = edge-säker: `html-react-parser` (htmlparser2, INGEN jsdom/DOMPurify → kraschar inte på Workers). Okänd modul-marker → inert span (kraschar aldrig sidan).
- Sanitizer vid SAVE (`sanitize.ts`, allowlist + `KNOWN_MODULE_TYPES` intakt) — inte per-render.
- Manifest = kanonisk slot-källa; `0040` reconcile + CI-guard (`salvia-slots-canonical.test.ts`, manifest↔migration). Typade regioner (text/image/logo), color/font ur slots (G1-provenance).
- goal-36 redan härdad på TVÅ punkter: **M1** (räkna FAKTISKA mallar, hantera trasiga explicit) + **M2** (mall-specifika proofs, inte smoke-test — "0 FAIL mot noll assertions = falsk grön").

## 🔴 CRASH-RISKER (måste fixas FÖRE den autonoma loopen)

1. **R1 — manuell vendor→mall-konvertering skalar inte** [CRIT]. Varje mall = handinlina ~700 rader HTML i `.ts` + skriv om asset-paths + handplacera `<corevo-module>` + manifest + proof. 4 gjorda; 100 = ~25× hand-fel (oavslutad tagg bryter literalen, fel path → trasig render). → **bygg en codemod** (vendor `index.html` → `.ts`-skelett + manifest-stub + asset-rewrite + strippa vendor-`<script>`). Utan den är "autonom loop" mest manuellt slit.
2. **R2 — `renderTemplate` saknar try/catch** [CRIT]. Trasig vendor-HTML → `html-react-parser` kan kasta → **SSR 500 på Workers**. (Okända *markers* är säkra; trasig *HTML* är det inte.) → wrap i try/catch + safe fallback; **parse varje mall EN gång vid build (author-time), faila DÄR — inte vid render.**
3. **R6 — bundla 100 HTML-literaler spränger Workern** [HIGH]. 700 rader × 100 i JS-bundeln → 10 MiB-gräns (gzip) + cold-start. → **lasta mall-HTML från R2/KV/DB vid request, inte ur bundlen.** (= "kompilera/lagra, rendera sen"-mönstret.) Behåll no-jsdom/no-`next/image`-regeln som lint.
4. **R4 — falsk-grön stub-proofs i skala** [HIGH]. M2 kräver riktiga proofs men enforcement = granskning; billigaste vägen vid 100 = copy-paste-stub som passerar utan assertion (62%-fällan). → **meta-assertion i `proof-kit.ts`: varje proof MÅSTE asserta ≥N region-id + booking-variant + ≥1 kanon-px/hex, annars failar proofet självt.** Gör stubbing mekaniskt omöjligt.
5. **R5 — 100 vendor-CSS-kit → global CSS-kollision** [HIGH]. Delade klassnamn (`.container`/`.btn`), `:root`-vars, resets. Scopas under `[data-tenant]` men svag scoping → visuell breakage som "renderar utan krasch" men failar design-trohet. → **verifiera scoping (prefix/shadow/iframe) + render-proof som assertar en COMPUTED style (ej bara DOM-närvaro) mot kanon-värde.**
6. **R7 — pattern-mismatch: salvia (React-layout) ≠ HTML-mallarna** [HIGH]. goal-36 §3 säger "följ `manifest/salvia.ts`" men salvia = option-1 React-layout (`SalviaLayout.tsx`), medan restoran/carserv/drivin/klinik = `renderTemplate`-vägen. En agent kan bygga fel sorts mall. → **peka loopen på `templates/restoran.ts` + `templates/carserv.proof.test.ts` som HTML-exemplar; säg EXPLICIT att salvia är en annan väg, kopiera den inte för katalog-mallar.**
7. **R8 — `validate_markers.mjs` saknas** [MED]. `render-bridge.tsx` lovar den "i S3"; finns ej i `5-Kod`. Felstavad `type` → tyst inert span, modulen försvinner, inget proof fångar. → **bygg validatorn (grep alla `.ts`, assert `type ∈ KNOWN_MODULE_TYPES`) + kräv att varje proof assertar den VÄVDA `data-corevo-module="X"` (live-modulen finns), inte bara markern.**
8. **R12 — hydration-mismatch från vendor-scripts/markup** [MED]. Inline `<script>`, icke-React-vänlig markup → server/klient-träd skiljer → hydration-krasch. → **strippa vendor-`<script>` i codemoden; render = ren funktion av (träd + tenant-data), inga timestamps/`Math.random`/`window` vid render; per-region error boundaries.**

## 🟠 SKALA / KORREKTHET (fixa under loopen)

- **R3 — generalisera canonical-guarden** (idag salvia-only) → parametriserad test över ALLA manifests (text/image/logo-nycklar == DB-slot-rader). En test, N mallar.
- **R9 — booking-variant per bransch** (gym/klinik/restaurang) — beror på **goal-40** (bransch-medveten booking). Massa-producera inte booking-mounts förrän goal-40 landat, ELLER proof assertar förväntad variant per mall.
- **R10 — out-of-band slot-drift** på de 26 importerade (generisk CMS-modell kvar). Reconcile var och en till sitt manifest (som `0040` för salvia) innan den aktiveras i render-bron.
- **R11 — licens/CC-BY-credit oenforced i kod** (beslut: behåll credit). → gör credit till en slot/region med per-mall `license`/`attribution`-fält i manifestet, assert närvaro i proof för CC-BY-källa. Manuell gate → mekanisk.

## 🔵 VERIFIERINGS-PIPELINE (mekaniskt 0 FAIL, à la theme-marketplaces)

Två-lager-gate (Shopify Theme Check + ThemeForest-QA-modellen):

1. **Statisk validering vid import** (billig, alla mallar): parse5-parse OK, regioner/manifest/defaults/modul-refs giltiga, marker-`type` känd. Faila → mallen släpps aldrig.
2. **Render-verify per mall × representativ tenant PÅ DEPLOYAD EDGE-preview** (Playwright `toHaveScreenshot` + assertions): 200, root mountar, **noll console-fel / ingen error-boundary**, varje region + `@module`-slot finns i DOM, screenshot-diff mot godkänd baseline. + **marknadströsklar** (Lighthouse perf ≥ 60 / a11y ≥ 90, med RIKTIGT innehåll — aldrig tom sektion). Valfritt: Chromatic/Percy för hostade baselines + flake-frys.

Edge-fällor: Worker 10 MiB (gzip) → lasta ej HTML ur bundlen · `nodejs_compat` datum/flagg-gatad → testa parser/sanitizer PÅ Workern, ej bara lokalt · bygg via icke-ö-path (`C:\tmp\kod`).

## Graceful-degradation-regler ("rendera eller degradera, krascha aldrig")

Varje region har en `default` → saknad/tom region renderar default, aldrig inget/throw. Saknad modul → statisk platshållare. Per-region error boundary (en trasig widget tar inte ner sidan). Parse-fel → loggad fallback, aldrig 500. Render = ren funktion av (träd + data) → ingen hydration-mismatch.

## Hur detta möter goal-36

goal-36 är redan bra på räkna-mallar (M1) + icke-triviala proofs (M2). De-risken lägger till: codemod (R1), try/catch + author-time-parse (R2), lasta-ur-storage (R6), anti-stub-meta-assert (R4), CSS-computed-style-proof (R5), peka-på-HTML-exemplar (R7), `validate_markers` (R8), generaliserad canonical-guard (R3), edge-render-verify (🔵). **Den autonoma loopen ska INTE köras förrän R1/R2/R6/R4 är på plats** — annars: en trasig mall kraschar batchen (R2), "autonom" blir manuellt slit (R1), eller 0 FAIL blir falsk-grön (R4).
